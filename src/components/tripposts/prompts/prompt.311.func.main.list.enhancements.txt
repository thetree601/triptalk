제목: 메인 목록 페이지 개선사항 - 기능 및 UI 명세

목표
- 메인 여행 게시글 목록 페이지에 날짜 검색, 제목 검색, 페이지네이션 그룹 이동, 게시글 삭제(휴지통 아이콘 + 확인 모달) 기능을 구현합니다.

범위
- 메인 목록 페이지 컴포넌트 (`src/components/tripposts/index.tsx`) 및 관련 hooks에 적용됩니다.
- 기존 검색, 필터, 페이지네이션 기능과 통합됩니다.

피그마 참고시 채널: ebl20vu4

요구사항

1) 날짜 검색 기능 구현
- 선택한 날짜 또는 날짜 범위로 여행 게시글을 필터링할 수 있어야 합니다.
- 날짜 입력은 단일 날짜 또는 시작일/종료일 범위 선택을 지원해야 합니다.
- 날짜 입력 변경 시 날짜 파라미터로 데이터를 다시 가져옵니다.
- 날짜 검색은 제목 검색 및 기타 필터와 함께 동작해야 합니다 (논리적 AND 연산).
- 날짜 필터는 공유 가능한 링크를 위해 URL 쿼리 파라미터와 동기화됩니다.
- 쿼리 변수: `dateFrom` (ISO 8601 형식), `dateTo` (ISO 8601 형식). 단일 날짜만 선택한 경우 두 값을 동일하게 설정합니다.
- 날짜 필드는 접근 가능하고, 키보드로 탐색 가능하며, 모바일 친화적이어야 합니다 (가능한 경우 네이티브 날짜 선택기 사용).
- 날짜 필터를 지우면 전체 목록이 복원됩니다 (다른 활성 필터는 유지).

2) 제목 검색 기능 구현
- 제목으로 게시글을 필터링하는 텍스트 입력을 구현합니다 (대소문자 구분 없음, 포함 검색).
- 입력 디바운스 처리 (예: 300ms)로 불필요한 재요청을 제한합니다.
- URL 동기화: `title=` 쿼리 파라미터가 현재 검색 값을 반영합니다.
- 목록 쿼리에 `titleContains` 변수를 추가합니다. 빈 값은 필터를 제거합니다.
- 제목 검색은 날짜 및 기타 필터와 논리적 AND 연산으로 결합됩니다.
- 제목 검색 결과가 없을 때 빈 상태를 표시합니다.

4) 페이지네이션 화살표 그룹 이동
- 좌우 화살표 동작 변경: 페이지별 이동(±1) 대신 5개씩 그룹 단위로 이동합니다.
- 페이지를 블록으로 그룹화: [1-5], [6-10], [11-15] 등.
- 오른쪽 화살표는 다음 그룹의 첫 페이지로 이동합니다 (예: 3페이지 → 6페이지).
- 왼쪽 화살표는 이전 그룹의 첫 페이지로 이동합니다 (예: [6-10]에서 [1-5]로 이동할 때 7페이지 → 1페이지).
- 마지막 그룹에 있을 때 오른쪽 화살표는 비활성화됩니다.
- 첫 번째 그룹에 있을 때 왼쪽 화살표는 비활성화됩니다.
- 현재 페이지는 여전히 상태 및 URL (`page=` 파라미터)에 저장됩니다.
- 개별 페이지 번호를 클릭하면 해당 정확한 페이지로 이동합니다.

5) 게시글 삭제: 휴지통 아이콘 hover 표시 + 삭제 확인 모달
- 각 게시글 행(Row) 우측(날짜 영역 우측 근처)에 휴지통 아이콘을 배치합니다.
- 아이콘은 데스크톱에서 행 hover 시에만 보이며, 포커스 상태에서도 접근 가능해야 합니다. 모바일/터치 환경에서는 행 포커스/탭 시 보이도록 처리합니다.
- 아이콘 리소스: `public/icons/delete.png` (디자인 참조용). Figma 참고: 채널 `daqshn4o`, 노드 ID `394:11398`.
- 아이콘 클릭 시 확인 모달을 띄웁니다. 메시지: "삭제하시겠습니까?" 버튼: [확인], [취소].
- GraphQL 스키마: `deleteBoard(boardId: ID!): ID!`
- [확인] 클릭 시 해당 게시글의 `boardId`로 `deleteBoard` 뮤테이션을 호출합니다.
- 성공 시 목록에서 해당 게시글을 제거합니다. 캐시/상태를 즉시 반영(낙관적 업데이트)하고 필요 시 목록을 재조회합니다.
- [취소] 클릭 시 아무 작업도 하지 않고 모달을 닫습니다.
- 오류 발생 시 사용자에게 에러 메시지를 표시하고, 낙관적 업데이트를 롤백합니다.

  레이아웃 재배치 (Figma 노드 `394:11398` 준수, 현재 순서 유지)
  - 열 순서는 현행 그대로 유지합니다: [번호] [제목] [작성자] [날짜] [휴지통 아이콘]. 날짜를 가운데로 이동하지 않습니다.
  - 우측 휴지통 아이콘 전용 고정 여백(가터)을 컴팩트하게 예약합니다: 아이콘 크기 24x24px, 가터/히트 영역은 28~32px(권장 32, 최소 28).
  - `grid-template-columns` 예: `auto minmax(0, 1fr) auto auto 28px`(또는 32px)로 마지막 컬럼을 아이콘 가터로 두거나, Flex에서 우측에 폭 28~32px 고정 컨테이너를 둡니다.
  - [제목]은 가변 폭으로 한 줄 말줄임 처리하여 [작성자]/[날짜] 및 우측 아이콘 영역을 침범하지 않도록 합니다.
  - [작성자], [날짜]는 최소 폭을 지정하고 넘침 시 말줄임 처리합니다(필요 시 날짜 포맷 축약 적용). 텍스트가 줄어들더라도 열 순서는 유지합니다.
  - 데스크톱에서는 단일 행 레이아웃을 유지합니다. 여백/간격을 미세 조정하여 우측 아이콘이 들어갈 공간을 확보합니다.
  - 모바일(<= 375px)에서도 큰 구조 변경 없이 간격/폭만 조정합니다. 필요한 경우에만 [작성자]를 2행으로 내릴 수 있으나 시각적 순서는 유지합니다.
  - 아이콘 히트 영역은 28~32px 범위 내에서 설정하고, 키보드 포커스 시에도 표시되도록 `:focus-visible` 스타일을 제공합니다.

수용 기준
- 날짜 검색이 날짜 범위 또는 단일 날짜로 게시글을 올바르게 필터링하며, URL과 동기화됩니다.
- 제목 검색이 제목 부분 문자열로 게시글을 필터링하며, URL과 동기화됩니다.
- 페이지네이션 화살표가 5페이지씩 그룹으로 이동합니다 (1-5 → 6-10 → 11-15 등).
- 게시글 행 우측에 휴지통 아이콘이 hover/포커스 시 나타납니다(모바일은 포커스/탭). 아이콘 클릭 시 확인 모달이 표시됩니다.
- [확인] 시 `deleteBoard(boardId)`가 호출되고, 성공 후 해당 게시글이 목록에서 사라집니다. [취소] 시 변화가 없습니다.
- 열 순서가 [번호] [제목] [작성자] [날짜]로 유지되며, 우측 아이콘 가터(28~32px) 추가에도 시각적 순서가 변하지 않습니다. Figma(노드 `394:11398`)와 시각적으로 일치합니다.

구현 참고사항
- 목록 렌더링 시 각 행에 hover 상태를 감지하여 아이콘을 표시/숨깁니다. 접근성을 위해 키보드 포커스 진입 시에도 아이콘이 노출되도록 합니다.
- 모달은 `src/commons/providers/modal/`의 패턴을 사용합니다. 컨텐츠: 메시지, [확인]/[취소] 버튼, Enter=확인, Esc=취소.
- GraphQL
  - 뮤테이션 정의 파일 추가/확인: 예) `src/lib/graphql/mutations/deleteBoard.ts`.
  - 문서 예: `mutation DeleteBoard($boardId: ID!) { deleteBoard(boardId: $boardId) }`
  - 호출부에서 `variables: { boardId }` 전달.
  - 캐시 업데이트: 삭제된 게시글을 현재 목록 캐시에서 제거하거나 목록 재조회(refetch) 수행.
  - 낙관적 업데이트: `optimisticResponse`로 해당 ID 제거를 즉시 반영, 실패 시 롤백.
- 레이아웃/CSS
  - Grid 또는 Flex 기반으로 우측 아이콘 28~32px 고정 가터와 좌측 컬럼들의 유연 폭을 설정합니다.
  - 제목/작성자/날짜 각각의 최소 폭 및 말줄임 처리를 통해 우측 아이콘 영역을 침범하지 않도록 합니다.
  - 모바일에서는 글자 수 축약, 간격 축소로 공간 확보하되 컬럼 순서는 유지합니다.
- UX
  - 모바일 터치 환경에서 hover가 없으므로 행 탭/포커스 시 아이콘 표시. 아이콘 히트 타겟 28~32px.
  - 삭제 직후 현재 페이지가 비게 되면, 이전 페이지로 이동하는 등의 페이지네이션 경계 케이스를 처리합니다.
  - 에러 토스트/알림 표시 및 재시도 옵션 고려.
- 테스트
  - 뮤테이션 호출 여부, 캐시/목록 업데이트, 에러 핸들링, 접근성(키보드 조작), 레이아웃 오버랩/잘림 방지, 컬럼 순서 유지 시나리오를 포함한 단위/통합 테스트를 추가합니다.
